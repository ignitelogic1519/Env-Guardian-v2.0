WHAT THIS PROJECT IS
-------------------------------------------------------------------------------
Env Guardian is a BYOD (Bring-Your-Own-Device) Mobile Device Management /
Zero-Trust security system. When an employee's personal Android phone enters a
defined "restricted zone" (a GPS geofence), the app enforces policy: it blocks
non-whitelisted apps, can cut internet with a local no-root VPN, verifies the
person's presence with a QR scan (static or rotating TOTP code), applies per-app
daily time limits, and logs every allow/block. When the phone leaves the zone,
full access is restored automatically.

Key principles:
- BYOD-friendly: NO factory reset, NO corporate enrollment. Runs as a normal
  app with granted permissions.
- Anti-theft: a device is permanently bound to its first owner; a lost/stolen
  phone cannot be re-registered by someone else.
- Privacy: enforcement ONLY happens inside the zone; nothing personal is
  collected outside it.
- Tamper-aware: disabling the enforcer, VPN or accessibility is detected and
  reported; devices heartbeat status/location/compliance in real time.

Tech stack: Flutter Android app (native enforcement) + Node.js/Express backend
+ PostgreSQL (Neon) + Render hosting. Plus a role-based static web admin console
(the /dashboard SPA, hosted separately) and a static marketing website (light,
professional SaaS design; the former AI chat widget has been removed).

Target industries: corporate/IT security, exams & education, healthcare, data
centres, R&D & manufacturing, government/defense, finance, call centres/BPO,
media & events.

-------------------------------------------------------------------------------
 /app  —  FLUTTER ANDROID CLIENT (the device agent)
-------------------------------------------------------------------------------
The mobile app installed on each employee device. Feature-first modular Dart
code plus native Kotlin services for the actual OS-level enforcement.

  app/lib/
    main.dart                          App entry point.
    app.dart                           Root app widget / routing.
    cloud_sync.dart                    Talks to the backend (register device,
                                       sync settings/geofence, push heartbeats,
                                       fetch whitelists & time-limit policies).
    core/
      background_service.dart          Foreground service loop: geofence check,
                                       enforcement trigger, heartbeat, per-app
                                       time limits (enforceTimeLimits). It writes
                                       in_restricted_zone; the Network Guard VPN
                                       itself is reconciled NATIVELY (see
                                       AppBlockerService) so it stops on zone exit
                                       even when the UI is closed.
      platform.dart                    Method-channel bridge to native Kotlin.
      theme/glass.dart                 Glassmorphism UI helpers.
      theme/neumorphic.dart            Neumorphism UI helpers.
    features/
      onboarding/admin_setup_screen.dart   First-run device registration. The
                                       required grants now run 0–9: the runtime
                                       permissions + Accessibility, the one-time
                                       Network Guard (VPN) consent, and the
                                       now-mandatory Usage Access, Notification
                                       Access and OEM Auto-start (acknowledgement).
                                       The device can't be sealed until all pass.
      command_center/command_center_screen.dart  Main dashboard/home screen.
                                       App-bar "Security Features" panel manages
                                       the optional protections (Usage Access,
                                       Notification Access, OEM Auto-start) at any
                                       time and shows the always-on Network Guard
                                       status (re-grant only; no in-app disable).
      map/map_tab.dart                 Shows restricted zone + device position,
                                       Inside/Outside badge.
      armory/armory_tab.dart           App whitelist / controls view.
      logs/log_tab.dart                Local allow/block event log.

  app/android/.../kotlin/com/example/env_guardian/   (NATIVE ENFORCEMENT)
    MainActivity.kt                    Method-channel host; permission prompts.
    AppBlockerService.kt               Accessibility service — detects & blocks
                                       non-whitelisted foreground apps. Honours a
                                       short "settings grace window"
                                       (enforcement_grace_until) so it stands down
                                       while the user is on a system settings page
                                       they were sent to (fixes the OEM auto-start
                                       screen closing instantly). Its 5s pulse also
                                       recomputes the effective whitelist natively
                                       (base from eg_base_whitelist minus apps that
                                       hit their per-app time limit, usage read via
                                       UsageStatsManager) and reconciles the Network
                                       Guard VPN — so whitelist changes, time limits
                                       and the VPN all work even with the UI closed.
    GuardianVpnService.kt              Local no-root VpnService — cuts internet
                                       for non-whitelisted apps (Feature B).
                                       START_NOT_STICKY + establishes only on an
                                       explicit ACTION_START, so it never resurrects
                                       itself (block-all) after an app kill. The live
                                       tunnel fd is held statically; closeTunnel()
                                       shuts it directly from anywhere (reconciler,
                                       manual button), so the VPN is guaranteed OFF
                                       outside the zone. All lifecycle events logged
                                       (logcat tag EnvGuardianVPN + in-app Logs tab).
                                       Re-established whenever the effective
                                       whitelist changes (e.g. an app hits its
                                       daily time limit → it also loses internet);
                                       onRevoke() sets a vpn_revoked tamper flag
                                       that the heartbeat now reports.
    GuardianNotifListener.kt           Notification-listener — pre-scan clean-
                                       state gate (Feature A).
    res/xml/accessibility_service_config.xml  Accessibility declaration.
    AndroidManifest.xml                Permissions & service registrations.

  Other folders (ios/, web/, linux/, macos/, windows/, test/) are the default
  Flutter scaffold — Android is the only actively targeted platform.

  App id (Feature H): com.envguardian.mdm (production).


-------------------------------------------------------------------------------
 /server  —  NODE.JS + EXPRESS BACKEND  (hosted on Render)
-------------------------------------------------------------------------------
REST API + PostgreSQL persistence. Configures zones, whitelists, time-limits;
receives device heartbeats; serves the admin QR page; proxies the website bot.

  server/src/
    index.js                 App bootstrap. Trust-proxy, helmet, CORS (allows
                             localhost + *.onrender.com / *.vercel.app /
                             *.netlify.app + CORS_ORIGINS env), morgan logging,
                             route mounting, /health, /qr page, 404/error
                             handlers, schema init on boot. Production safety
                             check refuses to start with insecure default
                             secrets.
    db/
      pool.js                PG connection pool (SSL auto-enabled off-localhost).
      initSchema.js          Creates all tables + seeds defaults on a fresh DB
                             (reports "Schema ready 42/42"). Idempotent: each
                             statement on its own connection, ALTER-backfills for
                             older databases.
      retention.js           Daily prune of NON-essential history (app_usage +
                             login_events) older than DATA_RETENTION_DAYS (10).
    middleware/
      auth.js                requireApiKey / requireJWT / requireAuth (either) +
                             requireRole gate; optional per-device-token
                             enforcement (Feature F).
    routes/
      auth.js                Admin login / JWT issue (+ /verify).
      agents.js              Device (agent) registration, heartbeats, remote
                             lock, compliance; QR current endpoint; plus the
                             /api/dashboard/* fleet listing + metrics APIs.
                             Device bound to first owner (anti-theft).
      settings.js            Geofence zone, whitelist, QR secret/mode, admin
                             password (GET needs auth + x-api-key).
      appUsage.js            App usage-stats ingestion + summaries.
      policies.js            Per-app daily time-limit policies + feature flags.
      deviceLogs.js          Live device enforcement-log stream — kept in a short
                             IN-MEMORY buffer (NOT persisted); console polls it.
      users.js               Console user management (create/edit/disable/delete;
                             admin only).
      aegis.js               Website chatbot LLM proxy (DORMANT — the website's
                             chat widget was removed; route kept for future use).
    utils/
      agentMetrics.js        Compliance-score / metrics helpers.

  server/public/
    qr.html                  Standalone admin page that displays the current
                             zone QR (asks for API key client-side, served
                             before helmet so its CSP doesn't block the QR lib).

  server/.env.example        Documents all env vars: PORT, NODE_ENV,
                             DATABASE_URL, JWT_SECRET, API_KEY, ADMIN_PASSWORD,
                             ENFORCE_DEVICE_TOKEN, LLM_API_KEY, LLM_MODEL,
                             CORS_ORIGINS. Never commit a real .env.

  AEGIS LLM PROXY (routes/aegis.js) — the AI chatbot backend:
    - POST /api/aegis/chat. Calls Anthropic (Claude) with a SERVER-SIDE key so
      no secret ever reaches the browser.
    - If LLM_API_KEY is unset, returns success:false so the widget falls back to
      its built-in FAQ brain (never breaks).
    - Guardrails in the system prompt: NEVER reveal secrets/passwords/keys/
      tokens/source code/bypasses; stay on-topic (product/business only); can't
      write code or perform actions — it only returns text answers about the
      product; every reply starts with an emotion tag [happy]/[love]/[neutral]/
      [confused]/[thinking].
    - In-memory per-IP rate limit (25 req/60s); message sanitizing (last 12,
      valid roles, capped length); 20s AbortController timeout; default model
      claude-haiku-4-5-20251001 (override via LLM_MODEL).


-------------------------------------------------------------------------------
 /dashboard  —  WEB ADMIN CONSOLE  (static SPA, role-based, host separately)
-------------------------------------------------------------------------------
An online admin console for the whole fleet: pure static HTML + CSS + vanilla JS
(no build step), hosted for free on a DIFFERENT site from the backend. It signs
in against POST /api/auth/login (JWT) and drives every admin operation through
the existing REST API. Built for large fleets (1000+ devices): every table
paginates with debounced search; the Overview surfaces the devices needing
attention first instead of rendering the whole fleet.

  dashboard/
    index.html               App shell (login + Intune-style nav rail/top bar).
    assets/
      config.js              EG_CONFIG: API_BASE, REFRESH_MS, COMPLIANT_AT
                             (API base can also be set at runtime on the login
                             screen).
      app.js                 SPA: auth, router, and all 9 pages.
      charts.js              Theme-aware chart rendering (metrics page).
      style.css              Two themes on one shell (dark glassmorphism / light
                             SaaS), switched by the sun/moon toggle; springy
                             staggered reveals; honours prefers-reduced-motion.
    README.md                Pages + full RBAC matrix.
    DEPLOYMENT.md            Free-hosting deploy guide (Netlify/Vercel/Pages/
                             Cloudflare/Render) + CORS notes.

  Pages (9): Home, Overview, Devices, Metrics, Policy Controller, QR Settings,
  Enrollment, Users & Roles, Settings. Live device logs stream into the per-
  device panel (the app pushes its native enforcement log; the console polls it).
  Roles come from users.role (admin / manager / viewer) and are enforced
  SERVER-SIDE (requireRole) — the UI only mirrors the gating.

  - Feature A: pre-scan clean-state gate (Notification Access).
  - Feature B: VPN network guard (block non-whitelisted apps' internet). ALWAYS-ON
    by policy: the one-time consent is captured during first-run setup (required to
    seal), then the guard AUTO-ACTIVATES on every zone entry with no further prompt.
    There is deliberately no in-app "off" switch; if the user disables the VPN in

    reported (vpn_revoked in the heartbeat). Lifecycle is reconciled NATIVELY in the
    accessibility service (starts on entry, STOPS on exit even with the UI closed)
    and tied to per-app time limits (over-budget apps lose internet too). BYOD limit:
    Android always allows disabling a VPN from
    system settings — enforcement is deterrent-grade (auto-recover + report), not an
    OS-level lock (that would need Device Owner, which this product avoids).
  - Feature C: auto-foreground prompt on zone entry.
  - Feature E: OEM auto-start helper + docs.
  - Feature F: per-device auth tokens (staged, opt-in enforcement).
  - Feature G: opt-in rotating (TOTP) QR codes.

  - Security: bind a device to its first owner; block re-registration.
  - UI: glassmorphism + neumorphism look with fonts and animations.

 HOW TO ACTIVATE / DEPLOY
===============================================================================
  - Backend: deploy /server on Render, set env vars (DATABASE_URL, JWT_SECRET,
    API_KEY, ADMIN_PASSWORD, and optionally QR_SECRET, DATA_RETENTION_DAYS,
    CORS_ORIGINS; LLM_API_KEY only if reviving the Aegis proxy).
  - Dashboard: set API_BASE in /dashboard/assets/config.js, then deploy the
    /dashboard folder to any free static host (Netlify/Vercel/Pages/Cloudflare/
    Render) — separately from the backend. See /dashboard/DEPLOYMENT.md.
  - Website: deploy /website folder to Netlify/Vercel/Pages/Render (static).
    (The AI chat widget was removed from the site; LLM_API_KEY is no longer
    needed for the website.)
  - App: build the Flutter Android APK from /app and install on devices.
